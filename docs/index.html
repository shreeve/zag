Zag is a systems language focused on clean, expressive syntax without giving up explicit control, predictable performance, or native compilation. It compiles to Zig, letting the Zig toolchain handle code generation, linking, and cross-compilation.

fun add a: i32, b: i32 -> i32
  a + b

sub main
  total = add 1, 2
  if total > 0
    print total
  else
    print 0
const std = @import("std");

fn add(a: i32, b: i32) i32 {
    return a + b;
}

pub fn main() void {
    const total = add(1, 2);
    if (total > 0) {
        std.debug.print("{any}\n", .{total});
    } else {
        std.debug.print("{any}\n", .{0});
    }
}

Features

  • Whitespace-sensitive, no semicolons, no braces
  • Expressions and routines produce values when used
  • Types optional in source, required by code generation
  • fun yields a value, sub is for effects
  • = for bindings, =! for constants — no let/var/const keywords
  • Prefix and postfix if, while, for, match
  • Structs, enums, error sets, tagged unions
  • Pipe |>, range .., power **
  • Compiles to readable Zig source

Quick Start

zig build                                    # build the compiler
./bin/zag --run test/examples/hello.zag      # compile and run
./bin/zag --compile test/examples/hello.zag  # emit Zig source
./bin/zag test/examples/hello.zag            # print S-expressions

Pipeline

Zag source → S-expressions → type resolution → Zig source → native binary.

The parser is generated by Nexus, a universal grammar engine shared across projects.
